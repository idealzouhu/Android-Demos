### 什么是悬浮按钮

悬浮按钮**（FloatingActionButton，FAB）** 是 Material Design 中立面设计这一思想的重要体现。

悬浮按钮（FloatingActionButton，FAB） 与普通按钮的主要区别在于视觉层级和交互优先级。

| 特性            | 悬浮按钮 (FAB)                     | 普通按钮                                 |
| --------------- | ---------------------------------- | ---------------------------------------- |
| 位置            | 悬浮在界面内容之上，通常右下角     | 嵌入在布局中，与其他元素平级             |
| 视觉层级        | 最高层级（elevation 8dp+），有阴影 | 普通层级（elevation 2-4dp）              |
| 功能定位        | 主要/高频操作（如新建、发送）      | 一般操作（如保存、取消）                 |
| 交互优先级      | 最突出，用户第一眼看到             | 按需使用，不强制吸引注意                 |
| Material Design | 标准组件，有圆形和扩展形态         | 多种样式（TextButton、OutlinedButton等） |



### 悬浮按钮的使用场景

当界面内容密集且有多个同等重要的操作的时候，FAB 会遮挡重要信息。这种情况下，通常是将主要操作集成到底部导航栏，滑动、长按等交互替换了显示按钮。

当应用有且只有一个主要操作（如"新建"），该操作使用频率较高，需要快速访问，不希望用户寻找的时候， 这种情况下就可以使用 FAB。

FAB 使用的场景主要有：

| 应用         | 使用场景 | 说明             |
| ------------ | -------- | ---------------- |
| Gmail        | 写邮件   | 主要操作突出显示 |
| Keep         | 新建笔记 | 核心功能快速访问 |
| Google 相册  | 上传照片 | 高频操作         |
| 部分记账应用 | 添加账单 | 单一主要操作     |
